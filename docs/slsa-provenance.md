# SLSA Provenance and Signature Verification

The v3.3 supply-chain workflow produces three independent forms of evidence for
the release zip built from `main` or a published release tag:

1. `our-skills-vX.Y.Z.slsa-provenance.json` is an inspectable in-toto Statement
   with the `https://slsa.dev/provenance/v1` predicate. It binds the artifact
   digest to the source commit, workflow, builder, invocation, timestamps, and
   release sidecars.
2. `our-skills-vX.Y.Z.sigstore.json` is a Cosign keyless bundle. Its short-lived
   signing certificate is issued from the GitHub Actions OIDC identity and its
   transparency-log proof is retained with the signature.
3. GitHub artifact attestation is generated by `actions/attest` and associated
   with the repository. It can be independently queried through GitHub's
   attestation API or CLI.

All external Actions in the workflow are pinned to full commit SHAs. The build
job has read-only repository access plus only the OIDC, attestation, and artifact
metadata write permissions needed to create evidence. Release upload runs in a
separate job and receives `contents: write` only for a `release.published` event.

## Verify the Cosign Bundle

Download the zip and matching `.sigstore.json` file from the same workflow run
or release. Replace the version and identity ref with the exact values shown by
that run:

```bash
cosign verify-blob our-skills-vX.Y.Z.zip \
  --bundle our-skills-vX.Y.Z.sigstore.json \
  --certificate-identity "https://github.com/2702207741-dev/agent-skills-pipeline/.github/workflows/supply-chain.yml@refs/heads/main" \
  --certificate-oidc-issuer "https://token.actions.githubusercontent.com"
```

For a tagged release, replace `refs/heads/main` with `refs/tags/vX.Y.Z`. The
identity and issuer checks are required; checking only the artifact digest does
not prove which workflow signed it.

## Verify the GitHub Attestation

```bash
gh attestation verify our-skills-vX.Y.Z.zip \
  --repo 2702207741-dev/agent-skills-pipeline
```

The verifier should report the repository, workflow identity, subject digest,
and Sigstore-backed attestation. Verification must fail for a modified zip, a
different repository identity, or a bundle copied from another workflow run.

## Inspect the SLSA Statement

The standalone statement is generated by:

```bash
python scripts/generate_slsa_provenance.py \
  --artifact dist-release/our-skills-vX.Y.Z.zip \
  --output dist-release/our-skills-vX.Y.Z.slsa-provenance.json
```

`python scripts/check_supply_chain.py` exercises the generator with fixed input
and rejects an incorrect statement type, predicate type, artifact digest,
source commit, builder id, or missing byproduct.

## Legacy local integrity sidecar

The retained v3.0.0 release predates the OIDC workflow. Its `.sig` file is a
deterministic SHA-256 integrity value over canonical local provenance. It can
detect modification when the trusted checksum is already known, but it does
not prove a maintainer or workflow identity. New CI-built bundles retain this
sidecar for backward compatibility and add the separate `.sigstore.json`
identity signature and GitHub artifact attestation. The project does not
retroactively describe the old `.sig` as a Sigstore signature.
