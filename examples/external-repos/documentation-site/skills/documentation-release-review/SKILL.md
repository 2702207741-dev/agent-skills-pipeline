# Documentation Release Review

## When to Use

Use this skill before publishing documentation that changes installation,
verification, compatibility, or release guidance.

## Workflow

1. Follow the primary navigation path from a clean reader entry point.
2. Run every safe local verification command shown to users.
3. Check that evidence links resolve to maintained artifacts.
4. Report broken instructions before approving publication.

## Verification Checklist

- [ ] Primary navigation reaches the release instructions.
- [ ] Commands are current and scoped to local data.
- [ ] Compatibility and rollback guidance are present.
